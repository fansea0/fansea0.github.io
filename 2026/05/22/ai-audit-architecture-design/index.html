设计案例解读 001 | AI审核服务插件化架构

项目：wespy-http-go | 分支：fd-5sp1-audit-music | 日期：2026-05-22

一、背景

在社交娱乐产品中，用户生成内容（UGC）的审核是保障平台安全的核心环节。传统的审核方式依赖人工或简单的关键词匹配，效率低且扩展性差。本项目采用 AI 大模型进行内容审核，并设计了插件化的多 Provider 架构，支持灵活切换不同的 AI 服务商。

二、核心设计：Provider 插件化

2.1 设计思路

AI审核服务通过统一的 provider 接口抽象了不同 AI 服务商（DeepSeek、Qwen、Doubao），每种服务商实现相同的接口方法，业务代码无需关心底层调用的是哪家模型。

// 定义统一的 Provider 接口
type AiCommonChecker interface {
    Check(ctx context.Context, input string) (string, error)
}

2.2 配置驱动切换

通过配置文件中的 UseProvider 字段决定使用哪个 Provider：

switch checkConf.UseProvider {
case DeepSeek:
    checker, err = deepseek.NewDeepSeekChecker(ctx, config)
case Qwen:
    checker, err = qwen.NewQwenChecker(ctx, config)
case Doubao:
    checker, err = doubao.NewDoubaoChecker(ctx, config)
}

2.3 优势

优势说明
可扩展新增 Provider 只需实现接口，无需修改核心逻辑
可切换通过配置变更就能切换 AI 服务商
可测试可以轻松 Mock 不同的 Provider 进行单元测试
解耦合业务逻辑与具体 AI 服务商解耦

三、Tool 工具系统

3.1 设计思路

AI 审核需要大量上下文信息（活动配置、礼物信息、礼包信息等），系统设计了 Tool 机制来收集这些信息，并注入到 prompt 中。

tools := provider.GetAvailableTools(checkConf.UseProvider)
for _, tool := range tools {
    result := tool.Execute(ctx, map[string]interface{}{
        "act_id": actId,
    })
    allToolResult += result
}

3.3 优势

  • 信息完整：AI 模型可以获取到完整的活动上下文
  • 职责分离：每种信息由专门的 Tool 负责收集
  • 灵活扩展：新增信息来源只需新增 Tool

四、上下文信息收集

审核前需要收集大量上下文信息，包括：

  1. 活动基础信息：actinfo.GetActById(actId)
  2. 礼物列表：遍历 actInfo.GiftList，调用 gift.GetGiftInfo()
  3. 盲盒配置：gift.GetBlindBoxConfig()
  4. 礼盒配置：gift.GetGiftBoxConfigByGiftId()
  5. 礼包信息：遍历 weconfig.GetGiftPkgInfo()

五、敏感词过滤（分层审核策略）

用户输入 -> 本地敏感词过滤 -> AI审核 -> 结果返回

好处：

  • 成本优化：本地过滤拦截明显违规的内容，减少 AI 调用
  • 速度快：敏感词过滤是本地操作，毫秒级响应
  • 精准度：AI 专注于复杂场景的判断

六、设计模式总结

设计模式应用场景
策略模式多 Provider 切换，通过配置选择不同算法
插件模式Tool 工具系统，可插拔的信息收集器
模板方法审核流程固定，具体实现可替换
责任链模式多层过滤（敏感词 -> AI审核 -> 结果处理）

七、值得学习的地方

  1. 接口抽象优于继承：用 interface 定义能力，而不是用继承构建类型体系
  2. 配置驱动行为：通过配置文件切换实现，而不是硬编码判断
  3. 信息收集分离：Tool 机制让上下文收集变得可扩展
  4. 容错设计：每个环节都有错误处理，单点失败不影响整体
  5. 成本意识：本地预过滤 + AI 审核的分层设计，优化成本

八、改进建议

  1. Tool 注册机制：目前 Tool 是硬编码获取，可以改为配置文件注册
  2. 缓存机制：上下文信息（如活动配置）可以加缓存，减少重复查询
  3. 异步处理：AI 审核是耗时操作，可以考虑异步队列处理
  4. 熔断降级：AI 服务不可用时，可以降级到本地规则审核
